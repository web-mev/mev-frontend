checkFeature table. This type of file describes the "features" of your data. In the genomics context, one example of a feature is a gene. Therefore, you could use this table to give additional information about each gene, such as alternative symbols, oncogene status, or similar. Each row contains information about a single gene. Note, however, that this concept is completely general and not restricted to information about genes or transcripts. {{ isTableShown['feature'] ? 'Hide example' : 'Show example' }}

gene2 2.2 0.01 0.03
gene3 3.3 3e-06 0.001

Sequence-based formats:

checkFasta is FASTA-format sequence files.
checkFastq is FASTQ-format sequence files. The most common format used for sequencing experiments.
checkAlignment (SAM/BAM) is BAM or SAM-format aligned sequence files. Typically the output of an alignment process.

JSON format:

checkJSON-format file is used for data that is not easily represented in a table-based format

General Format:

checkA general file is typically used to denote an unspecified type.
